Theory/Research


Shannon Entropy

Shannon entropy is a measure of uncertainty or randomness in a distribution. It quantifies how unpredictable or diverse a dataset is.

H = - Σ (pi * log2(pi))

  • H: Entropy of the distribution.
  • pi: Probability of the i-th outcome.

For example, in a system where all outcomes are equally likely, the entropy is highest. If one outcome dominates, entropy is lower.

Other Diversity Measures

Shannon entropy is one way to measure diversity, but there are other approaches:

  • Simpson's Index: Measures the probability that two randomly selected items belong to the same group:
    D = Σ (pi2)
  • Gini-Simpson Index: Focuses on the probability that two items belong to different groups:
    1 - D = 1 - Σ (pi2)
  • Rényi Entropy: A generalization of Shannon entropy with a sensitivity parameter α:
    Hα = 1 / (1 - α) * log2(Σ piα)

Primitive Roots

A primitive root modulo a prime number p is an integer g such that every integer a coprime to p can be expressed as a power of g modulo p. In other words, for any a where gcd(a, p) = 1, there exists an integer k such that:

gk ≡ a (mod p)

Key Properties
  • Modulo a prime: Primitive roots only exist modulo a prime number p or certain composite numbers.
  • Order: The smallest integer k such that gk ≡ 1 (mod p) is called the order of g.
  • Complete residue system: The powers of a primitive root modulo p generate all integers coprime to p.
Example

Let p = 7 and g = 3. To check if 3 is a primitive root modulo 7, compute the powers of 3 modulo 7:

  • 31 ≡ 3 (mod 7)
  • 32 ≡ 2 (mod 7)
  • 33 ≡ 6 (mod 7)
  • 34 ≡ 4 (mod 7)
  • 35 ≡ 5 (mod 7)
  • 36 ≡ 1 (mod 7)

The powers of 3 modulo 7 cover all integers coprime to 7 (1, 2, 3, 4, 5, 6). Thus, 3 is a primitive root modulo 7.

How to break the encryption?

// Decrypt message just using the frequency
let decryptMessage = function(frequencyOriginal, frequencyCypher){
	
	const alphabet = [
		'a', 'b', 'c', 'd', 'e', 'f', 'g', 'h', 'i', 'j', 
		'k', 'l', 'm', 'n', 'o', 'p', 'q', 'r', 's', 't', 
		'u', 'v', 'w', 'x', 'y', 'z'
	];
	let length = alphabet.length;
	let shift;

	let potentialShift = 0;
	let winner = 0;
	for(let i=0; i<length && winner == 0; i++){
		let matching = 1;
		for(let k=0; k<<length && matching == 1; k++){
			let currentCharOriginal = alphabet[k];
			//console.log("current char original: " + currentCharOriginal);
			let currentCharCypher = alphabet[ ( ((k+potentialShift)%26) + 26) % 26];
			//console.log("current char cyber: " + currentCharCypher);
			if(frequencyOriginal[currentCharOriginal] != frequencyCypher[currentCharCypher]){
				matching = 0;
			}
		}
		if(matching == 1){
			winner = 1;
			shift = potentialShift;
		}
		potentialShift++;
	}
	return shift;    
}
It’s easy find out which is the shift used in the encryption phase. Using this simple code we can do it in a very little computational time: for each character L in in the sorted alphabet it compares his frequency of the original text with L+SHIFT(potential shift) frequency of the cypher text, starting with SHIFT(potential shift) at 0. For a given potential SHIFT, if for all comparison it turns out that there is a match, then we have broken the encryption security.
