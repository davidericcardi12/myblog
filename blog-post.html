1. Theory (intro)


Population and Statistical Units

The population in statistics refers to the entire set of individuals, events, or objects that a study aims to analyze. Each individual member of this population is known as a statistical unit. For example, in a survey about the height of people, the population could be all adults in a country, and each person would be a statistical unit.


Distribution and Frequency

A distribution represents how the values of a variable are spread or distributed over a range of possible values. It helps describe how frequent certain values occur within the dataset.

Frequency refers to the number of times a value appears in a dataset:

  • Absolute frequency: The actual count of occurrences.
  • Relative frequency: The ratio of the frequency of a particular value to the total number of observations (expressed as a fraction or percentage).
  • Percentage frequency: The relative frequency multiplied by 100 to express it as a percentage.


Arithmetic Average

The arithmetic average, commonly known as the mean, is a measure of central tendency. It is computed by summing up all the values in a dataset and dividing by the number of values. Mathematically, it can be expressed as:

mean = (x₁ + x₂ + ... + xₙ) / n, where n is the total number of observations, and x₁, x₂,...,xₙ are the individual values.


Computational Issues: Floating Point Representation

When working with floating-point numbers in computers, precision problems can arise due to how these numbers are stored. One such issue is catastrophic cancellation, which occurs when subtracting two nearly equal numbers, resulting in a significant loss of precision. This is especially problematic in large datasets or when working with very small or very large numbers.


Numerical Solution (Knuth's Algorithm)

One approach to improving numerical stability in calculating sums and averages is using more precise algorithms, such as those described by Donald Knuth. His method reduces floating-point errors by using compensated summation. The idea is to account for small errors that accumulate during the summation process to avoid losing precision.

Example in Python:

An example implementation of the Kahan summation algorithm, a practical solution to minimize floating-point error, could look like this:


def kahan_sum(values):
	total = 0.0
	compensation = 0.0  # A running compensation for lost low-order bits.
	for value in values:
		y = value - compensation  # Compensate for lost bits.
		t = total + y  # Perform the addition.
		compensation = (t - total) - y  # Adjust compensation.
		total = t
	return total

This method helps reduce the rounding errors that can accumulate during summation in floating-point arithmetic, making the computation of means more accurate in large datasets.
